# Paradise Seafood Website — Claude Code Instructions

## Your Role
You are building a premium B2B seafood wholesale website for Paradise Seafood, London's leading fresh and frozen seafood wholesaler since 2007. You are simultaneously a world-class frontend engineer, a luxury brand designer, and a seafood industry expert.

## Quality Standard
Every page, component, and interaction must look like it was designed by a top London creative agency for a premium food brand. If any section looks generic, template-like, or "AI-generated" — stop and redo it with more personality, texture, and craft.

Think Natoora, Dishoom, or Ottolenghi's website quality — not a Shopify template.

## Before You Build Anything
1. Read BRIEF.md completely — it contains exact brand colours, typography, spacing, animation specs, and feature requirements
2. Read prototype.jsx — it's the approved homepage design, match its look and feel
3. Check the logo SVG files for exact brand colours (#0C1117, #B89B5E, #F0ECE2)

## Build Process
- Build ONE section at a time, not entire pages
- After each section, pause and let me review in the browser
- Use the exact hex values from BRIEF.md — never approximate colours
- Test every hover state, animation, and responsive breakpoint
- Mobile-first: design for phones first, then scale up

## Common Mistakes to Avoid
- Using #060a12 instead of #0C1117 for backgrounds
- Using #c9a84c instead of #B89B5E for gold
- Using #f0f4f8 instead of #F0ECE2 for cream text
- Flat backgrounds without subtle radial gradients
- Missing hover states on cards and buttons
- Generic section layouts without the Paradise Seafood personality
- Forgetting the four seafood icons from the logo as design elements
- Making text too small or too light to read comfortably
- Using emojis instead of SVG icons

## When in Doubt
- Re-read the relevant section of BRIEF.md
- Look at prototype.jsx for the approved design direction
- Ask me rather than guessing
